# User decisions from CONTEXT.md:
- Only QoS failures are retryable (PUBACK timeout, MQTT-level QoS 1 acknowledgment failures)
- Connection errors and broker errors are NOT retried
- Configurable max retries via YAML scenario config (e.g., retry: 3)
- Configurable backoff strategy via YAML (exponential, fixed, linear) and multiplier
- Skip to artifact on exhausted retries - write to failed_events.jsonl
- Run continues after retry exhaustion (individual message failure doesn't stop run)

# From 02-01-PLAN.md - MQTTPublishError:
This is the base exception from MQTTClient.publish()
We need to distinguish:
- Retryable: QoS acknowledgment failures
- Non-retryable: Connection errors, broker errors

# Backoff strategies:
- exponential: delay = base_delay * (backoff_multiplier ** attempt)
- fixed: delay = base_delay
- linear: delay = base_delay * attempt

# Artifact format (for failed_events.jsonl):
One JSON line per failed message:
{"trxId": "...", "payload": {...}, "error": "...", "retry_count": 3, "timestamp": "..."}

# Integration point:
WorkerPool._worker_publish_task() will wrap publish calls in RetryPolicy.retry()
</context>

2. Create BackoffStrategy enum:
   - EXPONENTIAL: delay = base_delay * (multiplier ** attempt)
   - FIXED: delay = base_delay
   - LINEAR: delay = base_delay * attempt

3. Create RetryPolicy class:
   - __init__(max_retries: int, strategy: BackoffStrategy, base_delay: float = 1.0, multiplier: float = 2.0)
   - Store configuration

4. Implement _calculate_delay() private method:
   - Based on strategy, calculate delay for given attempt number
   - Exponential: base_delay * (multiplier ** attempt)
   - Fixed: base_delay
   - Linear: base_delay * attempt

5. Implement async retry() method:
   - Accept func: callable (the publish function)
   - Accept *args, **kwargs for func
   - Try calling func(*args, **kwargs)
   - If RetryableError raised:
     - For attempt in range(max_retries):
       - Calculate delay using _calculate_delay(attempt)
       - await asyncio.sleep(delay)
       - Try func again
     - If all retries exhausted, raise MaxRetriesExceededError
   - If NonRetryableError raised: re-raise immediately
   - If other exception: re-raise
