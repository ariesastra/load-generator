# User decisions from CONTEXT.md:
- Global shared rate cap across all workers (e.g., 1000 msg/sec total)
- Token bucket algorithm - allows bursts within capacity
- Block/wait when rate exceeded (messages never dropped)
- Second-level precision for rate tracking

# Token bucket algorithm:
- Capacity: maximum number of tokens (burst size)
- Refill rate: tokens added per second
- acquire(): take 1 token, block if none available
- Refill happens periodically (every second or on each acquire)

# Example: rate_limit = 1000 msg/sec
- Capacity could be 1000 (1 second worth of messages)
- Refill rate: 1000 tokens/second
- Allows bursts of up to 1000 messages instantly
- Then throttles to 1000 msg/sec long-term

# Integration point:
WorkerPool will call await rate_limiter.acquire() before each publish
This ensures global rate limit across all workers

Implement src/loadgen/rate_limiter.py:

1. Create TokenBucketRateLimiter class:
   - __init__(self, rate_limit: int, capacity: Optional[int] = None)
   - rate_limit: messages per second
   - capacity: max tokens (defaults to rate_limit for 1-second burst)
   - _tokens: current token count (float for partial tokens)
   - _last_refill_time: timestamp of last refill (time.time())
   - _lock: asyncio.Lock for thread-safe token manipulation

2. Implement async _refill_tokens() private method:
   - Calculate elapsed time since last refill
   - Add tokens = elapsed_time * rate_limit
   - Cap at capacity
   - Update _last_refill_time

3. Implement async acquire() method:
   - Call _refill_tokens() to add any pending tokens
   - While _tokens < 1:
     - Calculate wait time = (1 - _tokens) / rate_limit
     - Sleep for wait_time
     - Call _refill_tokens() again
   - Decrement _tokens by 1
   - Return

4. Implement _refill_on_acquire() variant (simpler, no background task):
   - On each acquire(), calculate time since last acquire
   - Add tokens based on elapsed time
   - Block if insufficient tokens

5. Update tests/test_rate_limiter.py:
   - Test immediate acquire when tokens available
   - Test blocking when bucket empty
   - Test refill rate over time
   - Test burst behavior (capacity)
   - Use unittest.mock to patch time.time() for deterministic tests

Note: Use time.monotonic() for timing (not affected by system clock changes).
